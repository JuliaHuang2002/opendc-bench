# OpenDC-Bench 评测严谨化结果汇总

本报告汇总了「基线 + 显著性检验」评测的最终结果。所有模型在同一组预测窗口上评测（窗口起点由 `run_zeroshot.build_windows` 统一生成，各模型完全对齐），因此可做配对显著性检验。误差度量为平均绝对误差 MAE，预测步长 h ∈ {1, 6, 36, 144}（点数）。95% 置信区间由移动块自助法（block=50，n_boot=2000）给出；模型两两比较采用 Diebold–Mariano 检验（Newey–West HAC 方差，lag=h−1，并做 Harvey–Leybourne–Newbold 小样本修正），报告双侧 p 值。

评测数据：阿里 GPU trace 测试序列 N=1009 窗口；MIT SuperCloud 测试序列 N=1297 窗口。lookback=288。

## 四、结论（据实陈述，不夸大）

第一，基线并非陪衬。在阿里 GPU trace 上，最简单的持久性基线 naive-last 在短步长 h=1 反而显著优于预训练大模型（DM=4.05，p=0.0001）；到 h≥36，两者差异不再显著（p≈0.87–0.88）。这说明在该数据集上，基础时序大模型并未带来可被统计确认的增益，短步长上甚至更差。

第二，微调是否有用取决于数据集。在阿里数据上，LoRA 微调后的预测与预训练权重逐点完全一致（mean_diff=0，p=1.0），即微调没有产生任何可观测的改变；在 MIT SuperCloud 上，LoRA 微调在中等步长带来了幅度不大但统计显著的改进（h=6：p=0.003；h=36：p=0.031），h=1 与 h=144 不显著。

第三，在 MIT SuperCloud 上，基础大模型的价值得到确认。bolt-base 在 h≥6 全部步长上显著优于 naive-last（p<0.001），且在全部步长上显著优于滑动均值 ma-36。这与阿里数据的结论形成对照，指向一个可解释的差异：MIT 是 nvidia-smi 秒级实测功耗、序列更长更有结构，利于大模型发挥；阿里 trace 更接近调度回放、可预测结构有限。

一句话总结：模型的相对价值高度依赖数据集特性——盲目上大模型并非普遍最优，简单基线在部分场景仍是强对手，而微调的收益需要逐数据集用显著性检验确认，不能默认成立。

## 六、可复现说明

- 所有模型窗口对齐由 `run_zeroshot.build_windows` 保证（脚本内含 trues 对齐断言）。
